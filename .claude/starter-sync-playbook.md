# Sync Claude Code workflow from AI_Coding_Starter

**Goal:** update the workflow definitions (`.claude/`, the `.agents/` framework, `.gitignore`, `.mcp.json.example`) in this project, using https://github.com/mrozmk/AI_Coding_Starter as the source of truth. **Preserve everything project-specific** ‚Äî replace only workflow files.

**Assumption:** the project already has `.claude/` and `CLAUDE.md` (even an older version). If it doesn't ‚Äî stop and tell me this is a bootstrap, not a sync (in that case use GitHub "Use this template" instead of this prompt).

---

## Step 1: clone the starter into a temp directory

```bash
rm -rf /tmp/ai-coding-starter-sync
git clone --depth 1 https://github.com/mrozmk/AI_Coding_Starter /tmp/ai-coding-starter-sync
```

Save the starter's commit hash (`git -C /tmp/ai-coding-starter-sync rev-parse --short HEAD`) ‚Äî you'll use it in the final commit message.

## Step 2: classify the files

### Category A ‚Äî **overwrite wholesale from the starter** (workflow definitions, project-agnostic)

- `.claude/commands/*.md` ‚Äî all slash commands
- `.claude/agents/*.md` ‚Äî subagent definitions
- `.claude/skills/**` ‚Äî skill definitions (e.g. `jira/SKILL.md`)
- `.claude/templates/*.md` ‚Äî templates (e.g. `CLAUDE-template.md`, `README-template.md`)
- `.claude/hooks/*.sh` ‚Äî workflow hook scripts (`audit-append`, `check-deps`, `guard-commit`, `guard-memory`, `track-memory-read`). **Coupled with `settings.json`** ‚Äî the `hooks` block there references these scripts by path, so they must sync together. After copying, restore the executable bit: `chmod +x .claude/hooks/*.sh`.
- `.claude/README.md` ‚Äî **the framework guide**. Mind the path mapping: in the starter this guide lives as the **root `README.md`** (the GitHub template page); in a bootstrapped project it lives at `.claude/README.md` (moved there by `/setup:create-CLAUDE_MD`). Strategy: overwrite the project's `.claude/README.md` with the content of the starter's **root `README.md`**. Do **not** copy the starter's root README into the project root ‚Äî that would destroy the project's README (see category C).
- `.claude/starter-sync-playbook.md` ‚Äî **this playbook itself**. It is framework content, so it lives under `.claude/` (not `docs/`, which is project-owned / category C) precisely so it can self-update: each sync pulls the newest playbook from upstream. Overwrite wholesale.
- `.claude/workflows/*.js` ‚Äî `Workflow` orchestration scripts (e.g. `map-codebase.js`). Framework content ‚Äî overwrite wholesale.

### Category B ‚Äî **merge carefully**, show the diff and ask first

- `.claude/settings.json` ‚Äî the project may have its own permissions. Strategy: take the **union** of the `permissions.allow` / `permissions.deny` entries from the starter and the project. Do not remove project entries that the starter lacks. Show me the diff before writing.
- `.agents/memory/index.md` ‚Äî take `Quick Reference` and `Loader Convention` from the starter, but `When to Read` may have project-specific rows appended by `/setup:create-CLAUDE_MD`. Strategy: overwrite with the starter's structure, then restore the project rows (the ones the starter lacks).
- `.gitignore` ‚Äî append the entries missing from the starter (e.g. `.claude/audit.log`, `.mcp.json`, `.agents/memory/archive/`); **do not remove** project ones.
- `.mcp.json.example` ‚Äî overwrite if the starter has a newer version.
- `.claude/memory-domains.json` ‚Äî the `guard-memory` path‚Üídomain rules. Strategy: take the starter's `_examples` and any new keys, but **preserve project-filled `rules`** (the project's path regexes are project-specific). Show the diff; never wipe populated `rules`.
- `.editorconfig` ‚Äî append/merge missing keys from the starter; do not drop project-specific overrides.

### Category C ‚Äî **do not touch** (project content)

- `CLAUDE.md` ‚Äî project rules. If the starter has a new section structure, report it and propose a patch ‚Äî but do not overwrite automatically.
- `.agents/memory/architecture.md`, `project-brief.md`, `domain/*.md` ‚Äî regenerated from the project (`/setup:create-CLAUDE_MD`, `/maintain:refresh-brief`).
- `.agents/memory/errors.md`, `decisions.md`, `api.md`, `patterns.md` ‚Äî append-only, project history.
- `.agents/sources/`, `.agents/specs/`, `.agents/plans/`, `.agents/reference/`, `.agents/wiki/`, `.agents/memory/archive/` ‚Äî project content.
- `README.md` (root) ‚Äî the **project** README (generated by `/setup:create-CLAUDE_MD` at bootstrap). Never overwrite with starter content. You update the framework guide in `.claude/README.md` (category A).
- `LICENSE` ‚Äî the project may have relicensed; never overwrite.
- `CHANGELOG.md`, `docs/`, any source files ‚Äî project documentation and code.

1. Copy category A (new + changed) ‚Äî `cp -r` from `/tmp/ai-coding-starter-sync/` into the project
2. Perform the category B merge ‚Äî `settings.json` first, then `index.md`, then `.gitignore`
3. **Sanity check:**
   - All links in the copied commands resolve (`rg -o '\[.*?\]\(.*?\.md.*?\)' .claude/commands/`)

4. Clean up: `rm -rf /tmp/ai-coding-starter-sync`
