## Step 2: classify the files

### Category A тАФ **overwrite wholesale from the starter** (workflow definitions, project-agnostic)

- `.claude/commands/*.md` тАФ all slash commands
- `.claude/agents/*.md` тАФ subagent definitions
- `.claude/skills/**` тАФ skill definitions (e.g. `jira/SKILL.md`)
- `.claude/templates/*.md` тАФ templates (e.g. `CLAUDE-template.md`, `README-template.md`)
- `.claude/README.md` тАФ **the framework guide**. Mind the path mapping: in the starter this guide lives as the **root `README.md`** (the GitHub template page); in a bootstrapped project it lives at `.claude/README.md` (moved there by `/setup:create-CLAUDE_MD`). Strategy: overwrite the project's `.claude/README.md` with the content of the starter's **root `README.md`**. Do **not** copy the starter's root README into the project root тАФ that would destroy the project's README (see category C).

### Category B тАФ **merge carefully**, show the diff and ask first

- `.claude/settings.json` тАФ the project may have its own permissions. Strategy: take the **union** of the `permissions.allow` / `permissions.deny` entries from the starter and the project. Do not remove project entries that the starter lacks. Show me the diff before writing.
- `.agents/memory/index.md` тАФ take `Quick Reference` and `Loader Convention` from the starter, but `When to Read` may have project-specific rows appended by `/setup:create-CLAUDE_MD`. Strategy: overwrite with the starter's structure, then restore the project rows (the ones the starter lacks).
- `.gitignore` тАФ append the entries missing from the starter (e.g. `.claude/audit.log`, `.mcp.json`, `.agents/memory/archive/`); **do not remove** project ones.
- `.mcp.json.example` тАФ overwrite if the starter has a newer version.

### Category C тАФ **do not touch** (project content)

- `CLAUDE.md` тАФ project rules. If the starter has a new section structure, report it and propose a patch тАФ but do not overwrite automatically.
- `.agents/memory/architecture.md`, `project-brief.md`, `domain/*.md` тАФ regenerated from the project (`/setup:create-CLAUDE_MD`, `/maintain:refresh-brief`).
- `.agents/memory/errors.md`, `decisions.md`, `api.md`, `patterns.md` тАФ append-only, project history.
- `.agents/sources/`, `.agents/specs/`, `.agents/plans/`, `.agents/reference/`, `.agents/wiki/`, `.agents/memory/archive/` тАФ project content.
- `README.md` (root) тАФ the **project** README (generated by `/setup:create-CLAUDE_MD` at bootstrap). Never overwrite with starter content. You update the framework guide in `.claude/README.md` (category A).
- `CHANGELOG.md`, `docs/`, any source files тАФ project documentation and code.
